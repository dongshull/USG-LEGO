# USG-LEGO 后端接口文档

本文档详细描述了 USG-LEGO 项目的所有后端接口，包括请求方式、URL、参数、响应格式等信息，供前端开发人员对接使用。

## 基础信息

- **基础URL**: `http://localhost:8080` (默认端口)
- **请求格式**: JSON
- **响应格式**: JSON
- **认证方式**: API Key 认证

## 全局响应状态码

| 状态码 | 说明             |
|--------|------------------|
| 200    | 请求成功         |
| 400    | 请求参数错误     |
| 401    | 未授权/认证失败  |
| 403    | 禁止访问         |
| 404    | 资源不存在       |
| 500    | 服务器内部错误   |

## 接口列表

### 1. 健康检查接口

#### 接口说明
检查服务运行状态

#### 请求URL
```
GET /health
```

#### 请求参数
无

#### 响应示例
```json
ok
```

#### 响应说明
- 响应内容固定为 "ok"
- HTTP状态码: 200

### 2. 获取文件内容

#### 接口说明
获取指定路径的文件内容，支持文件流和JSON两种响应格式

#### 请求URL
```
GET /api/files
```

#### 请求参数
| 参数名 | 类型   | 必填 | 说明                                  |
|--------|--------|------|---------------------------------------|
| path   | string | 是   | 文件路径                              |
| api    | string | 否   | API密钥，访问私有文件时必需           |

#### 请求头
| 参数名          | 值                    | 说明                                       |
|-----------------|------------------------|--------------------------------------------|
| Accept          | application/json       | 指定返回JSON格式的文件信息                 |
| Accept          | application/octet-stream | 指定返回文件流（默认）                    |

#### 响应格式（文件流）
当请求头 Accept 不为 application/json 时返回文件流：

```
Content-Type: application/octet-stream
Content-Disposition: attachment; filename="filename.ext"

[文件内容]
```

#### 响应格式（JSON）
当请求头 Accept 为 application/json 时返回文件元数据：

```json
{
  "path": "/rules/default.list",
  "isDir": false,
  "fileInfo": {
    "name": "default.list",
    "isDir": false,
    "size": 85,
    "modTime": "2025-08-30T19:38:39.93346734+08:00"
  }
}
```

#### 响应格式（目录列表）
当请求路径为目录时返回目录内容列表：

```json
{
  "path": "/rules",
  "isDir": true,
  "files": [
    {
      "name": "default.list",
      "isDir": false,
      "size": 85,
      "modTime": "2025-08-30T19:38:39.93346734+08:00"
    }
  ]
}
```

#### 错误响应
```json
{
  "error": "错误信息"
}
```

#### 错误信息说明
| 错误信息           | HTTP状态码 | 说明                           |
|--------------------|------------|--------------------------------|
| Path parameter is required | 400        | 未提供path参数                |
| Invalid path       | 403        | 路径包含非法字符或路径遍历攻击 |
| File not found     | 404        | 文件或目录不存在               |
| API key required   | 401        | 访问私有文件未提供API密钥      |
| Invalid API key    | 401        | 提供的API密钥无效              |
| Server configuration error | 500 | 服务器配置错误                |
| Failed to access file | 500     | 服务器无法访问文件            |
| Failed to read directory | 500  | 服务器无法读取目录            |

### 3. 原始文件访问

#### 接口说明
直接访问原始文件内容

#### 请求URL
```
GET /raw/<文件路径>
```

#### 请求参数
| 参数名 | 类型   | 必填 | 说明                        |
|--------|--------|------|-----------------------------|
| api    | string | 否   | API密钥，访问私有文件时必需 |

#### 响应示例
```
Content-Type: text/plain; charset=utf-8

Raw file access placeholder
```

#### 响应说明
- 返回文件原始内容
- HTTP状态码: 200

### 4. 获取API密钥列表

#### 接口说明
获取所有API密钥列表（仅管理员）

#### 请求URL
```
GET /api/keys
```

#### 请求参数
| 参数名 | 类型   | 必填 | 说明       |
|--------|--------|------|------------|
| api    | string | 是   | 管理员密钥 |

#### 响应示例
```json
[
  {
    "id": "1",
    "token": "32字节随机字符串",
    "note": "密钥备注",
    "created_at": "2025-08-30T19:38:39Z"
  }
]
```

#### 响应说明
- HTTP状态码: 200

### 5. 创建API密钥

#### 接口说明
生成新的API密钥

#### 请求URL
```
POST /api/keys
```

#### 请求参数
| 参数名 | 类型   | 必填 | 说明       |
|--------|--------|------|------------|
| api    | string | 是   | 管理员密钥 |
| note   | string | 否   | 密钥备注   |

#### 请求体示例
```json
{
  "note": "用于iOS设备的密钥"
}
```

#### 响应示例
```json
{
  "id": "2",
  "token": "新生成的32字节随机字符串",
  "note": "用于iOS设备的密钥",
  "created_at": "2025-08-30T19:38:39Z"
}
```

#### 响应说明
- HTTP状态码: 200

### 6. 删除API密钥

#### 接口说明
删除指定的API密钥

#### 请求URL
```
DELETE /api/keys/:id
```

#### 请求参数
| 参数名 | 类型   | 必填 | 说明       |
|--------|--------|------|------------|
| id     | string | 是   | 密钥ID     |
| api    | string | 是   | 管理员密钥 |

#### 响应示例
```json
{
  "message": "API key deleted successfully"
}
```

#### 响应说明
- HTTP状态码: 200

## 权限控制说明

### 公共路径
- 以 `/public` 开头的路径为公共路径
- 公共路径下的文件可无需认证直接访问

### 私有路径
- 除公共路径外的所有路径均为私有路径
- 访问私有路径需要提供有效的API密钥

## 安全机制

1. **路径遍历防护**: 所有路径参数都经过 `filepath.Clean` 处理，并检查不能跳出 ROOT_DIR
2. **API密钥验证**: 私有文件访问需要提供有效的32字节随机字符串形式的API密钥
3. **访问控制**: 写操作和私有路径读操作都必须携带有效的API密钥

## 错误处理

所有错误响应都遵循以下格式：
```json
{
  "error": "错误信息"
}
```

## 示例请求

### 获取公共文件（无需API密钥）
```bash
curl "http://localhost:8080/api/files?path=/public/public.conf"
```

### 获取私有文件（需要API密钥）
```bash
curl "http://localhost:8080/api/files?path=/private.rules&api=您的API密钥"
```

### 获取文件元数据（JSON格式）
```bash
curl -H "Accept: application/json" "http://localhost:8080/api/files?path=/modules/module1.conf&api=您的API密钥"
```

### 获取目录列表
```bash
curl -H "Accept: application/json" "http://localhost:8080/api/files?path=/rules&api=您的API密钥"
```
